# Contributing to chrome-history-api

Thanks for your interest in contributing. This document covers the basics.

GETTING STARTED

1. Fork the repository on GitHub
2. Clone your fork locally
3. Install dependencies with `npm install`
4. Create a feature branch from `main`

DEVELOPMENT WORKFLOW

```bash
npm install
npm run build
npm test
```

Make sure the project builds and tests pass before submitting a pull request.

PULL REQUESTS

- Keep changes focused. One feature or fix per PR.
- Write clear commit messages that explain why, not just what.
- Add or update tests if your change affects behavior.
- Make sure `npm run build` and `npm test` both pass.
- Reference any related issues in the PR description.

REPORTING ISSUES

Use the issue templates provided in this repository. Include reproduction steps, expected behavior, and your environment details.

CODE STYLE

- TypeScript strict mode is enabled
- Keep functions small and focused
- Use meaningful variable and method names
- Avoid unnecessary dependencies
